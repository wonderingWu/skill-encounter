# 技能奇遇 (Skill Encounter) — 项目交接文档

> 最后更新：2026-05-22 | 代码仓库：https://github.com/wonderingWu/skill-encounter

---

## 一、项目是什么

**一句话：** 面向在校大学生的 AI 驱动能力成长平台。不是面试题库，不是 AI 聊天——是一个帮助学生在毕业前系统性提升六维核心能力的练习环境。

**核心差异化：** 不是「给你答案」，而是「让你练习」。不是「假装面试官」，而是「发现你对自己认知的偏差」。

---

## 二、目标受众

- **主要用户：** 大二、大三在读学生（2028-2029 年毕业）
- **所有专业通用：** 计算机/工科/文科/商科/设计/社科……不是只有 CS 学生需要
- **核心心理状态：** 对就业焦虑、对自己不了解、不知道学了能干什么、社恐/表达困难
- **用户不会说的需求：** 他们以为自己需要「面试技巧」，其实需要的是「了解自己缺什么」

---

## 三、产品核心机制

### 3.1 六维能力框架

| 维度 | 内涵 | 训练方式 |
|------|------|---------|
| 💬 表达影响力 | 口头/书面表达、说服力 | 即兴演讲、辩论、面试对话 |
| 🧩 逻辑分析力 | 结构化思考、问题拆解 | 费米问题、方案设计、产品分析 |
| 🪞 自我认知力 | 优势盲区识别、情绪管理 | 心理成长知识、MBTI/盖洛普引导 |
| 🤝 人际协作力 | 团队合作、冲突处理 | 群面模拟、职场关系场景 |
| 🤖 AI素养力 | AI工具使用、AI伦理 | AI产品思维、Vibe Coding引导 |
| 🌱 适应创新力 | 快速学习、跨界连接 | 产品设计、创业场景 |

### 3.2 用户流

```
用户进站 → ⚡30秒画像（年级+专业+3个担忧+六维自评）
         → 🎯 AI推荐对应场景
         → 💬 开始练习（AI角色扮演面试/辩论/演讲）
         → 📊 练习结束 → 六维AI评分 vs 自评对比
         → 🔄 看到差距 → 针对性再练 → 六边形逐步饱满
```

### 3.3 核心价值时刻

- **「自评 vs AI评」的差距** —— 用户以为自己表达力有 4 分，AI 只给了 2.5。这个认知冲击是用户回来的原因。
- **「跨会话的累积画像」** —— 练了 5 次之后，六边形从凹陷慢慢变饱满。ChatGPT 做不到这一点。

## 四、技术架构

### 4.1 当前栈

```
前端：原生 HTML/CSS/JS（单文件 SPA，约 700 行）
  ├─ 六维 SVG 雷达图（自绘，无依赖）
  ├─ 4 套一键切换主题（纯净/温暖/赛博/即刻风）
  ├─ 左侧分类+搜索栏
  └─ 30秒画像采集弹窗

后端：Python FastAPI
  ├─ LLM 服务层（OpenAI 兼容接口，自动适配 GLM/Claude/GPT）
  │   ├─ 双格式 Prompt 构建（XML 标签 for Claude/GPT，精简格式 for GLM）
  │   ├─ Token 追踪（每次调用记录 input/output）
  │   └─ 对话压缩 compaction（消息>10条时压缩早期对话）
  ├─ RAG 服务层（ChromaDB 向量检索，129条知识库）
  │   ├─ 检索缓存（LRU 128条，hash-based）
  │   └─ Just-in-time 检索（不在会话开始一次性注入）
  ├─ 评估器（六维评分 + 四维反馈）
  └─ 画像系统（内存存储，MVP阶段）

部署：Docker Compose（单容器 backend + chroma_data volume）
目标环境：GitHub Codespace
```

### 4.2 LLM 配置

```
当前使用：GLM-4-Flash（免费）/ GLM-4-air（付费，很便宜）
API Base: https://open.bigmodel.cn/api/paas/v4
Key 格式：智谱格式 {id}.{secret}（不是 OpenAI 的 sk-proj- 格式）

备选：
  - DeepSeek（有 key，余额不足）
  - OpenAI（有 key，没充值）
```

### 4.3 工作流设计原则（来自 Anthropic Context Engineering 指南）

- Context 是有限资源，有「注意力预算」——不是越多越好
- RAG 采用 Just-in-time 策略：按需检索，不在开始一次性注入
- 对话超过 10 条消息时触发 compaction（压缩早期对话为摘要）
- 评估前先压缩对话再打分（节省 40-60% token）
- 面试对话 temperature=0.4，评估 temperature=0.2

- ✅ 4 套一键切换主题
- ✅ 消息气泡（AI 白色靠左，用户蓝色靠右）
- ✅ Docker Compose 一键部署
- ✅ 在 Codespace 成功运行（GLM-4-Flash，0.0027元/2轮）

### 已知问题

- 对话超过 2-3 轮偶尔挂（GLM-4-Flash 能力有限，可能需要用 GLM-4-air）
- 六维评分在对话很短时精度有限（需要 3 轮以上才有意义）
- 画像存储是内存型，重启丢失（MVP 阶段可接受）

### 尚未实现（按优先级）

1. **跨会话画像持久化**（SQLite/Redis）—— 目前重启丢失
2. **六维历史成长曲线**—— 目前只存最新评分，没有时间序列可视化
3. **更多练习场景**—— 当前 6 个，后续可加「技术面试」「英语面试」「谈判模拟」
4. **多人对练模式**—— 当前的「轻社交」只体现在画像维度，没有真人互动
5. **微信小程序版本**—— 当前只有 Web 版
6. **课程/社团接入**—— 与高校选修课或社团活动关联的能力
7. **语音对话**—— 目前只有文字
8. **付费体系**—— 企业版/高级场景/无限练习

## 七、用户（wonderingWu）的偏好和要求

- **语言：** 中文为主，接受英文技术术语
- **风格：** 要深度、要系统性思考、反感仓促出代码
- **审核要求：** 有苏格拉底审查机制（见 AGENTS.md / SOCRATES.md），重大架构决策需审核
- **产品理念：**
  - "两只脚"：一只在校园（学生视角），一只在职场（用人需求）
  - "三维立体"：不只是面试技巧，要有心理成长+社会知识+AI素养
  - "正向但实际"：拒绝厚黑学，帮学生真正成长
  - "不能等跑代码才发现流程有问题"：SWOT+里程碑+风险预案要在编码前
- **RAG 持续完善：** 不追求一次完美，可以持续补充
- **开发阶段用最好模型，上线用性价比模型**
- **用户画像是核心差异化** —— 自评 vs AI评的对比，跨会话累积
- **场景语言要学生能懂** —— 不是"模拟面试"，是"投了简历没回音，是不是我不够好？"
- **聊天收集信息不可靠** —— 用下拉框、多选等传统 UI，不要 LLM 猜

## 八、环境与运行

### 本地开发

```bash
cd skill-encounter
cp .env.example .env   # 编辑填入 API key
docker-compose up --build
# 访问 http://localhost:8000
```

### Codespace 运行

```bash
# 创建 .env（用 GLM-4-Flash 免费额度）
cat > .env << 'EOF'
LLM_API_KEY=你的智谱key
LLM_API_BASE=https://open.bigmodel.cn/api/paas/v4
LLM_MODEL=glm-4-flash
LLM_TEMPERATURE=0.4
CHROMA_PERSIST_DIR=./chroma_data
CHROMA_COLLECTION_NAME=skill_encounter_knowledge
EOF

docker-compose up -d
# 端口 8000 需设为 Public
```

### 关键命令

```bash
docker-compose logs backend --tail=20   # 看日志
docker-compose restart backend          # 重启（不读新 .env）
docker-compose down && docker-compose up -d  # 完全重建（读新 .env）
docker system prune -af                 # 清理磁盘
```
